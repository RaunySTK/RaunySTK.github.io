Cadastro de Cliente

Atores

Cliente e Funcionário

Pré-condição

O cliente não deve ter cadastro no sistema.

Fluxo de eventos principal

  1. O funcionário seleciona "cadastar cliente".
  2. O cliente fornece o nome, endereço, telefone e CPF, ao funcionário.
  3. Se o cliente fornecer informações inválidas ao funcionáro, o sistema dará erro.
  4. O sistema pede para o funcionário verificar as informações novamente.
  5. Quando o cliente fornecer as informações válidas, o cadastro é marcado como concluído.

Fluxo de eventos alternativos

A qualquer momento o cliente pode solicitar ao funcionário que cancele o cadastro. O cadastro não será gravado no sistema.

Pós-condição

O cadastro deve esta gravado no sistema e marcado como confirmado.
